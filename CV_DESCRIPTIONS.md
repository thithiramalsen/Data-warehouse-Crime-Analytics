Project Title: LAPD Crime Analytics Data Warehouse & Cube | Data Engineering, BI & Reporting

Technologies: Python (Pandas), T-SQL, SSIS, SSAS (Multidimensional), Excel, Power BI, Visual Studio, CSV

Short (1-line)
- Project lead: Built an SSIS ETL pipeline, SSAS cube, interactive Power BI dashboards and Excel reports to model and analyze LAPD crime data for multi-dimensional trend analysis.

Medium (CV bullet set)
- Designed and implemented SSIS ETL packages to ingest, clean, and transform LAPD crime records into a dimensional data warehouse (facts + dimensions).
- Developed an SSAS multidimensional cube with measures and hierarchies enabling fast slice-and-dice analysis by date, area, weapon, premise, and victim.
- Built interactive Power BI dashboards and Excel pivot reports to visualize crime trends, hotspots, and weapon/premise breakdowns for stakeholders.
- Created Python utilities (Pandas) to extract and normalize lookup tables and documented reproducible dataset acquisition to keep repository lightweight.

Detailed (expanded bullets suitable for LinkedIn or a project section)
- Led end-to-end delivery of a crime analytics solution: crafted SSIS pipelines to stage raw CSV data, apply validation and transformations, and populate dimension and fact tables using incremental/accumulating load patterns.
- Implemented an SSAS cube design (dimensions, measure groups, aggregations) to support interactive reporting and cross-filtered analysis of crime patterns across geography and time.
- Developed interactive Power BI reports and Excel workbooks with calculated measures, slicers, and pivot tables to enable non-technical stakeholders to explore trends and export insights.
- Built Python tooling (Pandas) to extract, standardize, and publish lookup tables (crime codes, weapons); added clear dataset sourcing so collaborators can reproduce the ETL locally without committing large files.
- Maintained repository hygiene by excluding large artifacts and build outputs via `.gitignore`, ensuring the project remains shareable while protecting large data assets.
